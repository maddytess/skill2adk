# skill2adk

Convert [agentskills.io](https://agentskills.io/specification) / Anthropic
[`SKILL.md`](https://github.com/anthropics/skills) skills into Escher ADK
packages, using Claude (Opus 4.7) for field-level inference and strict pydantic
validation for output correctness.

## What it does

Given a skill directory like:

```
skills/pdf/
  SKILL.md
  reference.md
  forms.md
  scripts/
    check_bounding_boxes.py
    fill_fillable_fields.py
    ...
```

it produces a registration-ready ADK package:

```
out/domain.documents.pdf/
  agent_registry.json
  skills/
    documents.pdf_form_fill.json
  tools/
    documents.check_bounding_boxes/
      documents.check_bounding_boxes.json
      documents.check_bounding_boxes.py     ← copied verbatim from source
    documents.fill_fillable_fields/
      ...
  guardrails/
    documents.never_fabricate_form_values.json
    ...
```

## Installation

```bash
python -m venv .venv
source .venv/bin/activate
pip install -e .
```

## Configuration

Create a `.env` file in the project root:

```bash
CLAUDE_API_KEY="sk-ant-..."
SKILL2ADK_MAX_RETRIES=3       # optional, default 3
```

The CLI loads `.env` automatically. `--api-key` and `--retries` flags override the
environment.

## Usage

### Local skill directory

```bash
skill2adk /path/to/skills/pdf --domain documents --out ./out
```

### A parent directory containing many skills

The ingester recursively finds every `SKILL.md` under the source path and bundles
them into one ADK package:

```bash
skill2adk ./my-skills/ --domain security --out ./out
```

### A GitHub URL

Public or private (the converter shells out to `git clone`, so it inherits your
local git auth). Both repo and subpath URLs work:

```bash
skill2adk https://github.com/anthropics/skills/tree/main/skills/pdf \
  --domain documents --out ./out

skill2adk https://github.com/anthropics/skills --domain general --out ./out
```

### All flags

```
skill2adk SOURCE --domain DOMAIN [--out OUT] [--retries N]
                  [--api-key KEY] [--env-file PATH] [--quiet]

  source       Local skill directory, parent dir, or github URL.
  --domain     Domain name (e.g. 'security'). Becomes domain.{domain}.{name}.
  --out        Output directory; package goes in <out>/<agent_id>/. Default: ./out
  --retries    Per-call retry budget. Falls back to SKILL2ADK_MAX_RETRIES, then 3.
  --api-key    Override CLAUDE_API_KEY / ANTHROPIC_API_KEY from .env.
  --env-file   Path to .env (default: ./.env).
  --quiet      Suppress per-call usage logging.
```

## How it works

The full ADK package schema is too complex for Anthropic's structured-output
grammar compiler in one shot, so inference is split across multiple **strict**
calls. Each call uses `client.messages.parse(output_format=PydanticModel)` with
adaptive thinking and `effort=high`.

```
1. PLAN          (1 call)   PackagePlan — IDs and source mapping
2. AGENT REGISTRY (1 call)   AgentRegistry
3. PER SKILL     (2 calls)   SkillBody + ExecutionPlan, merged into Skill
4. PER TOOL      (1 call)    Tool — one per executable script in source skill
5. PER GUARDRAIL (1 call)    Guardrail
```

Each call:
- has its own strict pydantic schema (small enough for the grammar compiler);
- shares a frozen system prompt (rules + spec + reference packages) marked as a
  prompt-cache breakpoint, so repeat calls hit cache and pay ~0.1× input cost;
- retries up to `SKILL2ADK_MAX_RETRIES` times on validation/parse failure,
  replaying the error text back to the model as feedback;
- aborts immediately on grammar-too-large errors (those need a code split, not a
  retry).

Tool source files (`.py`) are **not** generated by the LLM. The model emits
`source_filename` pointing back into the source skill; the writer copies the
file verbatim into the output package.

## Strict validation

Every emitted document is also re-validated post-hoc against pydantic models
with `extra="forbid"`. The same models load all 38 reference documents in
`escher-adk-packages/` cleanly, so anything that parses also satisfies the
ADK package format.

```python
from skill2adk.models import AgentRegistry, Skill, Tool, Guardrail
import json
AgentRegistry.model_validate(json.load(open("out/.../agent_registry.json")))
```

## Project layout

```
skill2adk/
  __init__.py
  models.py        # Pydantic models for the ADK format + inference plan
  ingest.py        # Local dir / GitHub URL → SkillCorpus
  inference.py     # Per-document strict Claude calls with retry + cache
  generator.py     # InferredPackage → on-disk package layout
  cli.py           # Console entry point
spec/
  package_format.md       # Source of truth for the ADK package format
escher-adk-packages/      # Reference packages (4 domains, used as cached examples)
```

## Limitations

- One CLI invocation = one ADK package = one domain. Multi-domain bundling is
  not supported; run the converter once per domain.
- The model decides which scripts in a skill become tools. Pure helpers
  (utilities, test fixtures) without a clear callable interface are skipped.
- Tool source files larger than 64KB are truncated when sent to the model. The
  full file is still copied to the output package — the truncation only affects
  what the model sees during inference.
- GitHub URLs are cloned shallowly (`--depth 1`) into a temp directory.
